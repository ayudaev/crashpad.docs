Handles exceptions raised by exception_raise_state().

This behaves equivalently to a catch_exception_raise_state() function used with exc_server().

There is no destroy_request parameter because, unlike CatchExceptionRaise() and CatchExceptionRaiseStateIdentity(), the request message is not complex (it does not carry the thread or task port rights) and thus there is nothing to destroy.
